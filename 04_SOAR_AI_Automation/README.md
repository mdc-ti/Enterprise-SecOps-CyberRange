# Phase 4: Local AI-Powered SOC Alerting Pipeline (Wazuh SIEM + n8n + Ollama + Telegram API)

## 🤖 Overview & Automated Architecture

The final core phase of this Cyber Range focuses on evolving the security architecture from a legacy log aggregator into an intelligent, autonomous alerting and orchestration system. By introducing a local Large Language Model (LLM), the infrastructure constructs a dynamic **Security Orchestration, Automation, and Response (SOAR)** pipeline that mimics the workflow of a Tier 1 SOC Analyst.

Raw, complex JSON alerts generated by the Wazuh SIEM are programmatically intercepted, normalized, and evaluated by an on-premises artificial intelligence model. The AI contextualizes the threat and outputs a highly concise, natural-language executive summary, which is instantly pushed to security personnel via a secure Telegram Bot channel. 

**Absolute Privacy Baseline:** 100% of the ingestion, parsing, and LLM inference is executed locally within the self-hosted environment. Zero data is shared with external cloud providers, guaranteeing total data sovereignty.

---

## 📐 Laboratory Orchestration Blueprint

* **Detection Core (Wazuh Manager):** Ingests raw endpoint events on Ubuntu Server (`192.168.1.144`) and triggers integration daemons on high-severity thresholds.
* **Orchestration Workflow Engine (n8n):** Self-hosted node cluster deployed via Docker containers, handling JSON parsing, webhook endpoints, and conditional logic routing.
* **Local Inference Server (Ollama):** Local compute layer running the optimized **Llama 3 (8B Parameters)** model optimized for rapid technical contextualization.
* **Notification Gateway (Telegram Bot API):** Secure API channel configured with customized HTTP access tokens and restricted chat group constraints.

---

## 🛠️ Technical Milestones & Core Implementations

### 1. Custom Webhook Integration
Configured the Wazuh integration daemon (`ossec.conf`) to actively listen for specific alert levels (e.g., Level 10+ critical alerts). Upon match, the engine serializes the threat telemetry and sends an HTTP POST request to the n8n webhook listener.

### 2. Prompt Engineering & SOAR Logic Mapping
Designed advanced system prompts inside n8n to constrain the Llama 3 model behavior. The prompt strips away raw JSON formatting and forces the local AI to structure its threat assessment into three distinct bullet points: **Threat Vector**, **Affected Assets**, and **Immediate Remediation Steps**.

### 3. API Hardening & Secure Alert Routing
Leveraged Telegram's Bot infrastructure to establish an instantaneous alerting plane. The n8n engine executes an automated REST API call (`sendMessage`) targeting verified internal Chat IDs, ensuring that critical perimeter alerts are received on engineers' mobile devices in less than 5 seconds from initial endpoint compromise.
