# Project 4: Local AI-Powered SOC Alerting Pipeline (Wazuh + n8n + Ollama + Telegram)

## Overview
The critical purpose of this project was to evolve the SIEM architecture from a simple log aggregator into an intelligent, autonomous alerting system. By introducing a local Large Language Model (LLM), the pipeline acts as an automated Tier 1 SOC Analyst.

Raw alerts generated by Wazuh are intercepted, normalized, and injected into a local instance of Llama 3. The AI analyzes the threat context and generates a concise, natural-language executive summary, which is immediately pushed to a mobile device via a secure Telegram Bot. All processing is done 100% locally to guarantee absolute data privacy and sovereignty.

## Laboratory Architecture
* Detection Core (Wazuh Manager): Ubuntu Server (192.168.1.144). 
* Orchestration Engine (n8n): Deployed via Docker (macOS host). 
* Inference Server (Ollama): Local macOS environment running the llama3 model (8B parameters).
* Notification Gateway (Telegram API): Secure Bot endpoint authorized via HTTP Access Token.

## Project Phases and Technical Milestones
1. Webhook Implementation and Telemetry Ingestion: Configuration of the Wazuh integration daemon to forward specific alert thresholds.
2. Local Artificial Intelligence Integration (SOAR): Prompt Engineering and dynamic injection of Wazuh JSON variables.
3. Mobile Alerting and API Security: Telegram Bot Provisioning.
4. Troubleshooting and Sanitization: JSON Syntax Hardening.

## Executive Summary
* Security Orchestration and Automation (SOAR): Successfully designed and deployed a multi-node automation pipeline using n8n.
* Applied Artificial Intelligence & Privacy: Integrated Ollama and the Llama 3 model entirely on-premises. Ensured total data sovereignty.
* Incident Response and Alerting: Configured secure API integrations using Telegram's Bot infrastructure.
